### OQ-006 — Rate units and quarterly conversion conventions (global)
**Status: RESOLVED FOR PROJECT IMPLEMENTATION (2026-07-17) — via user-confirmed annualized-rate convention.** Within the proposed 2026 suite, the source states ÷4 annual→quarterly conversion explicitly only for trading NII data construction (PDF p. 225) and inside the securities coupon formulas (Coupon Rate/4; N/360 for hedge legs); current-suite footnote 54 (subordinated debt) separately states "coupon rates are always stated as annualized rates" with an annual-rate ÷4 (PDF p. 155) — supports the annualized-rate resolution; identified in the Step 5 review (2026-07-17). For the deposit-expense models ("multiplying the modeled rate by the average balance", PDF pp. 209, 214), the calculators (Eqs A39, A43, A48), and other-borrowing (Eq A53(1)), whether rates are annualized (requiring ÷4 for a quarterly flow) is not stated — **this source-side absence remains true and is preserved**. Loans state "interest income is quarterly compounded" (assumption 3, PDF p. 184) without defining the rate basis. **Resolution (PROJECT IMPLEMENTATION DECISION — USER CONFIRMED):** all interest rates used in this project are annualized rates; when an annualized projected rate is used to calculate one quarter's interest expense, the annualized rate is divided by four (simple nominal quarterization, not effective compounding). For domestic time deposits: Expense(b,t) = AverageBalance(b,t) × Rate(b,t) / 4, with the Equation A44 recursion remaining entirely in annualized-rate units — division by four occurs only in the final quarterly-dollar expense calculation. The PPNR source itself does not explicitly state the quarterly conversion for this component; the resolution is project implementation context, not a Fed statement. Where the source states its own conversion (securities /4; hedge N/360; trading-NII ÷4), the source statement governs. See `research/source-briefs/interest-expense/deposits/ie_dom_time_dep.source-brief.md` §7.9 (PID-6).
